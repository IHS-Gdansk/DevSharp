THE CONFERENCE

Some things you should know about Dev Sharp Conference

DevSharp is a free conference dedicated to those which want to nurture their knowledge and skills as a developers. Six international speakers, associated with the IT, who are happy to share their experience and pass information focused on the latest technologies. Our meeting is also an amazing opportunity to ask questions to our experienced speakers and interact with the other participants.

This is the second edition of DevSharp, a conference organized by IHS Markit. Last year's presentations were organized only for employees, but the success and great acceptance of the participants resulted in the creation of place for all interested developers.

Join us! Learn with us!

Meet people we will learn from

Mark Seemann

Mark Seemann helps programmers make code easier to maintain. His professional interests include functional programming, object–oriented development, software architecture, as well as software development in general. Apart from writing a book about Dependency Injection he has also created several Pluralsight courses, written numerous articles and blog posts about programming, and created or contributed to various open source projects.

Karl-Henrik Nilsson

Experienced developer and Microsoft .NET MVP that have written code for anything from cellular network base stations to websites. He is a passionate software developer at tretton37 and spends a somewhat obsessive amount of his free time building smarter devices."If you ever need to see a man about building a smarter toaster - I'm that guy"

Mark Rendle

Founder of Rendle Labs, creators of CloudLens and providers of consultancy and training in next-generation .NET and cloud computing. Mark has been doing this stuff for more than quarter of a century, and he still hasn't got bored. The rumours that Mark starts rumours about himself, to make himself look better than he is, are definitely not true.

Tomasz Heimowski

Tomasz is a passionate developer whose main areas of interest are F# and Functional Programming in general. As a Software Engineer at IHS Markit he tackles problems from various domains. In his free time, apart from contributing to OSS projects and learning new tech stuff, he enjoys lifting weights as well as dancing.

Andrzej Dostatni

Andrzej Dostatni is a Polish former ice dancer. He is a six-time Polish national champion with partners Honorata Górna and Małgorzata Grajcar and 1988 Olympian. In professional career first toured with Disney On Ice and later with Darlin Baker performed in Sun Valley Ice Show and competed in US Open. Dostatni was one of the first ISU Technical Specialists in Ice Dancing and performs this function today. He married Russian former ice dancer Elena Khalyavina in September 2009.

Christopher Klug

Chris Klug is an adrenaline loving, problem solver/creator that loves building things. He spends his days writing code and solving problems for clients at a company called Novatrox in Stockholm, and his nights sleeping. Except for the days he spends at conferences, talking about doing things he is passionate about, and the days he fills with mountain biking, kiteboarding or RC helicopter flying. For the last 15-something years, he has been solving problems for customers. But if he gets to choose, he'll go to the beach and kitesurf. Every time!

  • 11:40 | Tomasz Heimowski

    XSLT - Use the right tool for the job

    XSLT is a mature Domain Specific Language focused on processing XML documents. It is however (un)famous for quickly becoming overcomplicated and hard to reason about. Lots of developers usually get frightened before, frustrated during and freak afterwards writing code in this language. In this talk I’ll show how to use the best of XSLT features in order to make the code more maintainable. By reaching to its functional parts, I want to prove that this DSL doesn’t have to be so scary, but instead can become really beneficial.

  • 13:40 | Christopher Klug

    AngularJS – Tips from the Trenches

    When you start building Angular applications, you generally go from “WTF” to “this is pretty cool and useful” very fast. Unfortunately, you also tend to go on to “damn, this is messy”, “oh, I wonder how I am going to test this” and “why do I spend so much time debugging stupid mistakes”, in a fairly short time. Luckily, there are ways to get around this, and stay at “Angular makes me so productive”.

    In this talk, you will see some things you can do to make the code easier to work with, easier to test and less prone to simple mistakes. Things that the speaker, Chris Klug, has learnt over the last couple of years, while working with Angular projects for clients. Small tips from the real world. Tips that can hopefully make you more productive and alleviate some of the common problems that arise while building applications with Angular.

    Building and Running ASP.NET Core Applications with Docker

    .NET Core 1.0 and ASP.NET Core 1.0 have been released! All this means that you can run your ASP.NET Core MVC applications in Docker containers on Linux machines, which is a brilliant way to run anything.
